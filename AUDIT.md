## 5. Incohérences détectées

### 🔴 Critiques

**Domaine dans robots.txt incorrect**
- `robots.txt` pointe vers `https://www.capvietnam.fr/sitemap.xml`
- Le vrai domaine est `https://www.blog-capvietnam.fr/`
- Impact : Google ne trouve pas le sitemap depuis robots.txt

**URLs propres non configurées dans .htaccess**
- Le sitemap utilise des URLs sans `.php` (ex: `/visa-vietnam-francais-guide-2026`)
- Le .htaccess actuel ne redirige que `/` → `blog-capvietnam.php`
- **Aucune règle de réécriture générale** pour `slug` → `slug.php`
- Impact : les URL du sitemap risquent de retourner 404 sur le serveur

**Email non professionnel**
- `email-professionnel@tutanota.com` apparaît dans 7 fichiers (footer.php, mentions-légales, confidentialité, cookies, blog-capvietnam.php)
- Devrait être remplacé par `contact@blog-capvietnam.fr` (ou email réel professionnel)

**Prix guide à 1 €**
- Mentionné dans le titre de `guide-cap-vietnam-2026.php`, dans la nav, dans le footer
- Le plan prévoit de passer à 9 € — à valider avec toi avant modification

### 🟠 Importants

**Date d'arrivée incohérente**
- `guide-cap-vietnam-2026.php:217` : "Français à Hanoï depuis 2022"
- `travailler-en-ligne-vietnam.php:123` : "Télétravailleur depuis Hanoï depuis 2022"
- `transferer-argent-vietnam-wise.php:162` : "Utilisateur de Wise depuis 2021"
- **Question à valider :** Quelle est la date réelle d'arrivée/installation à Hanoï ?

**Compteurs hardcodés**
- `articles-capvietnam.php:128` : "19 articles" (hardcodé — devra être mis à jour manuellement à chaque article)
- `articles-capvietnam.php:100` : "19" dans le hero stats (idem)
- `article-type-capvietnam.php:145` et `articles-capvietnam.php:143` : "💬 12 commentaires" sur la carte article en une — faux, aucun commentaire visible

**Stat "∞ Phở mangés"**
- `a-propos-capvietnam.php:188` : peu crédible, à remplacer par une vraie métrique

**Photo profil sur domaine externe**
- 20 occurrences de `anthony-bouillon.com/profil2.jpg`
- Dépendance externe : si ce domaine tombe, toutes les photos d'auteur disparaissent
- À télécharger en local dans `assets/img/profile/`

**Schema BreadcrumbList et FAQPage absents des articles**
- Présents uniquement dans le template `article-type-capvietnam.php`
- Tous les articles publiés (19) n'ont ni BreadcrumbList ni FAQPage dans leur schema
- Perte de rich snippets potentiels dans Google

### 🟡 Mineurs

**`article-type-capvietnam.php` a le même titre que l'article visa**
- `page_title` = "Visa Vietnam pour un Français…" — c'est le template, il devrait avoir un titre canonique différent ou être exclu du crawl (robots noindex)

**`merci-guide.php` schema vide `{}`**
- Ne provoque pas d'erreur mais inutile — à supprimer ou remplacer par `null`

**Fichier HTML statiques dupliqués**
- Pour chaque `.php`, il existe un `.html` correspondant (`a-propos-capvietnam.html`, etc.)
- Probablement des exports statiques — risque de contenu dupliqué si accessibles publiquement

**Logo PNG de 314 KB**
- Le logo dans le `<nav>` est textuel CSS (pas d'img) ✅
- Mais `logo-entreprise.png` est référencé dans les schemas comme favicon/publisher logo
- 314 KB pour une image de logo est excessif — SVG ou optimisation nécessaire

## 6. Performance

### Scripts externes
Par page de blog, les ressources bloquantes ou tier-3 sont :
- Google Fonts (2 appels `preconnect` + 1 `<link>`) — impact faible avec preconnect
- Formspree (soumission de formulaire seulement, non bloquant)
- PayPal sur `guide-cap-vietnam-2026.php` (script externe bloquant potentiel)
- Aucun Google Analytics / tag manager — bon signe pour la vie privée et la perf

### CSS inline
- Tout le CSS est inline dans `<style>` dans chaque page via `$page_extra_css`
- Pas de feuille de style externe à mettre en cache entre pages
- Impact sur les pages vues multiples : le CSS est re-téléchargé à chaque page

### Images sans dimensions
- **0 image** n'a `width` et `height` définis → CLS potentiel sur toutes les pages
- Les images Unsplash sont chargées en taille originale sans définition de format (`auto=format&fit=crop` parfois absent)

### Absence de preconnect Unsplash
- `images.unsplash.com` est utilisé sur toutes les pages mais aucun `<link rel="preconnect">` n'est déclaré dans header.php

### Format d'image
- Tout est en JPEG via Unsplash — aucun WebP, aucun AVIF, aucune directive `<picture>`
